# MiniDo V1 需求设计文档

日期：2026-05-06

## 1. 产品概述

MiniDo 是一个极简个人待办事项微信小程序，服务于「一个人管自己的事」的场景。它解决 Notion 太重、现有待办工具功能堆砌、上手成本高的问题。

V1 的产品核心是：打开即记、今天优先、记完就走，同时保证用户数据尽量不丢失。

### 1.1 产品定位

一个人使用的极简待办小程序。

不做协作，不做复杂项目管理，不做知识库。MiniDo 只帮助用户记录、提醒、完成和回顾自己的事情。

### 1.2 产品口号

少一点管理，多一点完成。

### 1.3 V1 总方案

MiniDo V1 = 极简个人待办 + 轻详情图片 + 单条提醒 + 静默云同步 + 轻标签周报。

## 2. 目标用户与核心场景

### 2.1 目标用户

- 需要管理个人事务的普通用户
- 觉得 Notion、飞书、多维表格、复杂待办工具太重的用户
- 希望快速记录、按今天处理、每周简单复盘的人
- 有写周报需求，但不想维护复杂项目系统的人

### 2.2 核心场景

1. 用户想到一件事，打开小程序，3 秒内记录一条待办。
2. 用户今天打开小程序，看到今天该处理、已经逾期、已经提醒但未完成的事情。
3. 用户完成一件事，快速勾选完成，列表自动变清爽。
4. 用户为某条任务补充详情文字和图片，避免时间久了忘记上下文。
5. 用户设置单条任务提醒，到时间后收到微信订阅消息。
6. 用户周末或周五生成本周摘要，复制后用于写周报。
7. 用户换设备后，任务和图片可以通过静默云同步恢复。

## 3. 设计原则

### 3.1 极简优先

所有功能都应服务于「记事、提醒、完成、回顾」。如果某个功能不能明显帮助用户完成这些动作，V1 不做。

### 3.2 本地优先

用户操作必须先本地生效。网络慢、云同步失败、图片上传失败、提醒授权失败，都不能阻塞任务保存。

### 3.3 不打扰用户

V1 不做传统登录流程，不要求用户授权头像、昵称、手机号。系统通过微信小程序能力静默获取用户标识，并用于同步和提醒。

### 3.4 今天优先

首页默认打开「今天」视图。MiniDo 的第一任务不是展示所有功能，而是告诉用户现在该处理什么。

### 3.5 回顾轻量化

周报能力只做摘要和复制，不做复杂图表、趋势分析、效率评分。

## 4. V1 功能范围

### 4.1 V1 包含

- 待办创建、编辑、完成、取消完成、删除
- 标题、详情文本、详情图片、截止日期、提醒时间
- 每条任务最多 3 张图片
- 首页顶部双 Tab：今天 / 全部
- 底部快速输入框
- 创建时可展开高级选项
- 独立任务编辑页
- 已完成任务自动折叠
- 删除二次确认
- 单条任务自定义提醒
- 免登录体验
- 本地优先保存
- 静默 openid 云同步
- 图片云存储同步
- 轻标签
- 全部页按标签筛选
- 周报摘要
- 一键复制周报文本

### 4.2 V1 不包含

- 协作、分享、评论
- 项目/清单分组
- 标签颜色、层级、标签管理页
- 优先级
- 子任务
- 重复任务
- 日历视图
- 图表化统计
- 趋势分析
- 效率评分
- 富文本编辑
- 除图片外的附件
- 视频、语音
- 回收站
- 手机号登录
- 头像昵称授权登录
- Web 端
- 完整多端账号体系
- 固定每日提醒
- 批量提醒

## 5. 信息架构

### 5.1 页面结构

V1 页面包括：

- 首页
- 任务编辑页
- 周报页

### 5.2 首页

首页承担两个核心职责：

- 查看今天该处理的事情
- 快速创建新任务

首页顶部为双 Tab：

- 今天
- 全部

默认打开「今天」。

首页底部固定快速输入框，用于快速创建标题任务。输入框旁边提供展开按钮，用于打开高级选项。

### 5.3 任务编辑页

任务编辑页用于查看和修改完整任务信息。

可编辑字段：

- 标题
- 详情文本
- 详情图片
- 截止日期
- 提醒时间
- 标签

删除入口放在任务编辑页内，点击后需要二次确认。

### 5.4 周报页

周报页从「全部」页右上角进入。

周报页用于查看本周任务摘要，并复制周报文本。

## 6. 任务数据设计

### 6.1 任务字段

每条任务包含以下字段：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| id | string | 是 | 本地生成的任务 ID |
| openid | string | 否 | 静默获取后写入，用于云同步 |
| title | string | 是 | 任务标题 |
| detail | string | 否 | 任务详情文本 |
| imageFileIds | string[] | 否 | 云存储图片 fileID，最多 3 个 |
| localImages | object[] | 否 | 本地图片临时信息，用于上传前展示和重试 |
| tags | string[] | 否 | 轻标签，最多 3 个 |
| dueDate | string | 否 | 截止日期 |
| reminderAt | string | 否 | 提醒时间 |
| reminderStatus | string | 否 | none / pending / enabled / failed |
| completedAt | string | 否 | 完成时间 |
| deletedAt | string | 否 | 删除时间，软删除使用 |
| createdAt | string | 是 | 创建时间 |
| updatedAt | string | 是 | 更新时间 |
| syncStatus | string | 是 | synced / pending / failed |

### 6.2 字段规则

- 标题必填。
- 详情文本可选，纯文本，不支持富文本。
- 图片可选，每条任务最多 3 张。
- 标签可选，每条任务最多 3 个。
- 截止日期可选。
- 提醒时间可选。
- 提醒时间不能早于当前时间。
- 删除采用软删除，写入 deletedAt。
- V1 不提供回收站和恢复入口。

## 7. 首页与列表规则

### 7.1 首页布局

首页使用顶部双 Tab：

- 今天
- 全部

底部固定快速输入框。

任务列表上方展示未完成任务，底部折叠展示已完成任务。

### 7.2 列表项展示内容

列表项展示：

- 完成勾选入口
- 标题
- 截止日期标识
- 提醒时间标识
- 详情摘要一行
- 图片标识
- 标签

列表页不展示大图，避免页面变重。

### 7.3 已完成任务

- 已完成任务自动折叠在底部。
- 用户可以展开查看。
- 已完成任务可取消完成。
- 已完成任务不出现在今天的未完成区域。

### 7.4 删除

- 删除入口在任务编辑页。
- 删除前二次确认。
- 确认后任务立即从列表消失。
- 底层写入 deletedAt。

## 8. 今天视图规则

### 8.1 进入今天视图的任务

未完成任务满足以下任一条件时，进入今天视图：

- 截止日期是今天
- 截止日期早于今天
- 提醒时间是今天
- 提醒时间早于当前时间，且任务未完成

同一条任务只显示一次。

### 8.2 今天视图排序

今天视图未完成任务排序优先级：

1. 已逾期未完成
2. 提醒时间已过但未完成
3. 今天提醒
4. 今天截止
5. 其他进入今天的任务

同一优先级内排序：

1. 有提醒时间的任务，按提醒时间升序。
2. 有截止日期的任务，按截止日期升序。
3. 无明确时间的任务，按 updatedAt 倒序。

超过 3 天的逾期未完成任务默认折叠为「更早逾期」入口，用户可展开查看。折叠只影响展示，不改变任务状态，不隐藏今天或最近逾期任务。

已完成任务进入底部折叠区。

## 9. 全部视图规则

### 9.1 全部视图展示

全部视图展示所有未删除任务。

排序规则：

1. 未完成任务在前
2. 未完成任务按截止日期优先
3. 无截止日期任务排在后面
4. 已完成任务折叠在底部

### 9.2 标签筛选

全部视图支持按标签筛选。

标签规则：

- 每条任务 0-3 个标签
- 标签为纯文本
- 新建和编辑页可添加或移除标签
- 输入标签时展示最近使用的 3-5 个标签，支持点击快速选择
- 最近使用标签按当前用户本地和云端任务记录计算，不需要单独标签管理页
- 不做标签颜色
- 不做标签层级
- 不做标签管理页

## 10. 新建任务流程

### 10.1 快速新建

1. 用户在底部输入框输入任务标题。
2. 点击添加。
3. 系统立即创建任务。
4. 任务先保存到本地。
5. 后台尝试同步到云端。

快速新建只要求标题。

### 10.2 高级新建

用户点击底部输入框旁边的展开按钮，打开高级选项。

高级选项包括：

- 详情文本
- 图片
- 截止日期
- 提醒时间
- 标签

保存后，任务立即本地生效。图片上传、云同步、提醒授权在后台处理。

### 10.3 校验规则

- 标题不能为空。
- 图片最多 3 张。
- 标签最多 3 个。
- 提醒时间不能早于当前时间。

## 11. 编辑任务流程

### 11.1 进入编辑页

用户点击任务列表项，进入独立任务编辑页。

### 11.2 可编辑内容

编辑页可修改：

- 标题
- 详情文本
- 图片
- 截止日期
- 提醒时间
- 标签

### 11.3 保存规则

- 用户修改后保存，先写入本地。
- 本地保存成功后，返回或停留均可。
- 后台同步云端。
- 同步失败不阻塞保存。
- 未同步状态不在首页打扰展示，只在编辑页显示。

## 12. 图片能力

### 12.1 图片范围

V1 支持：

- 拍照添加图片
- 从相册选择图片
- 每条任务最多 3 张图片
- 详情页缩略图展示
- 点击图片预览大图

V1 不支持：

- 视频
- 文件
- 语音
- 图片标注
- 图片裁剪
- 图片编辑

### 12.2 图片上传

图片使用微信云开发 CloudBase 云存储。

流程：

1. 用户选择或拍摄图片。
2. 小程序先本地展示图片。
3. 后台压缩图片。
4. 压缩后调用云存储上传。
5. 上传成功后获得 fileID。
6. 任务数据保存 fileID。
7. 换设备时通过 fileID 展示图片。

图片压缩规则：

- 图片上传前必须在小程序端压缩。
- 单图大小建议控制在 200KB-500KB，最终阈值以开发测试后的清晰度和成本平衡为准。
- 压缩失败时可尝试上传原图；如原图上传失败，按图片上传失败处理。

### 12.3 图片上传失败

- 图片上传失败不阻塞任务保存。
- 编辑页显示图片上传失败状态。
- 用户可在编辑页重试。
- 首页不展示上传失败状态，避免打扰。

## 13. 提醒能力

### 13.1 提醒范围

V1 支持单条任务自定义提醒时间。

V1 不支持：

- 每日固定提醒
- 重复提醒
- 批量提醒
- 周期任务提醒

### 13.2 提醒流程

1. 用户设置提醒时间。
2. 保存任务时请求微信订阅消息授权。
3. 用户授权成功，系统保存提醒。
4. 到提醒时间后，服务端发送订阅消息。

### 13.3 授权失败

如果用户拒绝授权或关闭授权：

- 任务照常保存。
- 不设置提醒。
- 提示文案：任务已保存，提醒未开启。

### 13.4 授权频次

系统应记录用户订阅消息授权结果，尽量减少重复弹窗。

规则：

- 用户设置提醒时间时才请求订阅消息授权。
- 如用户选择「总是保持以上选择」或系统返回稳定授权结果，后续同类提醒尽量复用该结果。
- 如授权状态不可复用，仍按微信小程序订阅消息机制在用户保存提醒时请求授权。
- 授权频次优化不能阻塞任务保存。

### 13.5 服务端提醒失败

如果服务端创建提醒失败：

- 任务照常保存。
- 提醒状态标记为 failed。
- 编辑页展示失败状态。
- 用户可以重试。

### 13.6 完成或删除后的提醒

- 用户完成任务后，不再发送后续提醒。
- 用户删除任务后，不再发送后续提醒。

## 14. 存储与同步

### 14.1 体验策略

V1 采用：

本地优先 + 静默 openid 云同步。

用户不需要登录，不需要授权头像昵称手机号。

### 14.2 本地保存

本地保存完整任务数据，保证：

- 打开即用
- 离线可用
- 弱网可用
- 操作即时反馈

### 14.3 静默身份

小程序后台静默获取 openid。

openid 用于：

- 云数据库任务同步
- 云存储图片归属
- 订阅消息提醒

### 14.4 云同步

同步规则：

- 新建任务先本地保存，再云同步。
- 编辑任务先本地保存，再云同步。
- 完成任务先本地保存，再云同步。
- 删除任务先本地软删除，再云同步。
- 云同步失败不阻塞本地操作。
- 未同步状态只在编辑页展示。
- 软删除数据保留 30 天后可进行物理清理。
- 物理清理不提供用户可见入口，由云函数或后台维护任务执行。

### 14.5 换设备恢复

用户在新设备打开小程序后：

1. 系统静默获取 openid。
2. 从云数据库拉取任务。
3. 合并本地任务。
4. 通过图片 fileID 展示云端图片。

### 14.6 冲突处理

如本地和云端出现同一任务冲突，V1 使用 updatedAt 较新的版本。

补充规则：

- 如任一端任务已完成，最终状态优先保留完成状态。
- 如任一端任务已软删除，且 deletedAt 晚于另一端 updatedAt，最终状态优先保留删除状态。
- 图片上传状态不覆盖任务文本内容；图片同步失败只影响对应图片。

## 15. 周报摘要

### 15.1 入口

周报入口放在「全部」页右上角。

首页仍保持今天 / 全部双 Tab，不新增本周 Tab。

### 15.2 周报范围

默认展示自然周数据。

自然周口径：

- 本周完成数：completedAt 落在本自然周内的任务数量。
- 本周未完成数：截至本周结束仍未完成，且创建时间不晚于本周结束的任务数量。
- 完成率：本周完成数 / (本周完成数 + 本周未完成数)。
- 按标签分组的已完成任务：仅统计 completedAt 落在本自然周内的任务。
- 待跟进任务：未完成且截止日期或提醒时间不晚于本周结束的任务。

周报包含：

- 本周完成数
- 本周未完成数
- 完成率
- 按标签分组的已完成任务
- 待跟进任务
- 逾期未完成任务提醒

### 15.3 周报复制

周报页提供一键复制周报文本。

示例：

```text
本周完成 12 项，未完成 3 项，完成率 80%。

【工作】
- 完成 MiniDo V1 需求梳理
- 整理客户反馈

【学习】
- 阅读微信小程序云开发文档

待跟进：
- 报销材料提交
- 体检预约
```

### 15.4 周报不做

V1 不做：

- 图表
- 趋势分析
- 效率评分
- 自定义报表模板
- 导出 PDF
- 自动发送周报

## 16. 状态与提示

### 16.1 任务保存

任务保存成功后，优先给用户即时反馈。

### 16.2 提醒授权失败

提示：

任务已保存，提醒未开启。

### 16.3 图片上传失败

编辑页展示：

图片上传失败，可重试。

### 16.4 同步失败

编辑页展示：

待同步或同步失败。

首页不展示同步状态，避免干扰。

## 17. 权限与隐私

### 17.1 用户授权

V1 不请求：

- 头像
- 昵称
- 手机号

V1 可能请求：

- 订阅消息授权
- 相册权限
- 相机权限

### 17.2 数据原则

- 任务数据属于用户个人数据。
- openid 仅用于识别用户、同步数据和发送提醒。
- 图片按用户和任务归属存储。
- 不做公开分享。
- 不做协作访问。

## 18. 技术实现建议

### 18.1 小程序端

建议使用微信小程序原生框架。

核心模块：

- 任务本地存储模块
- 云同步模块
- 图片选择与上传模块
- 提醒授权模块
- 今天视图计算模块
- 周报生成模块

### 18.2 云端

建议使用微信云开发 CloudBase。

云端能力：

- 云数据库保存任务数据
- 云存储保存图片
- 云函数获取 openid
- 云函数处理订阅消息发送

### 18.3 数据集合

建议集合：

- tasks
- reminder_jobs

图片使用云存储，不直接存入数据库。

## 19. 验收标准

### 19.1 基础待办

- 用户可以快速创建仅标题任务。
- 用户可以创建包含详情、图片、截止日期、提醒时间、标签的任务。
- 用户可以编辑任务所有 V1 字段。
- 用户可以完成和取消完成任务。
- 用户删除任务时有二次确认。
- 删除后任务从列表消失。

### 19.2 今天视图

- 今天截止的未完成任务显示在今天。
- 已逾期未完成任务显示在今天。
- 今天提醒的未完成任务显示在今天。
- 提醒时间已过但未完成的任务显示在今天。
- 同一任务不重复显示。
- 已完成任务进入折叠区。

### 19.3 全部视图

- 全部视图展示所有未删除任务。
- 未完成任务在前。
- 有截止日期任务按截止日期优先。
- 无截止日期任务排在后面。
- 已完成任务折叠。
- 可以按标签筛选。

### 19.4 图片

- 每条任务最多添加 3 张图片。
- 图片可拍照或从相册选择。
- 图片上传云存储。
- 图片上传失败不阻塞任务保存。
- 编辑页可看到上传失败并重试。

### 19.5 提醒

- 用户可为单条任务设置提醒时间。
- 提醒时间不能早于当前时间。
- 授权成功后提醒生效。
- 授权失败后任务照常保存。
- 授权失败提示：任务已保存，提醒未开启。
- 完成或删除任务后不再发送提醒。

### 19.6 同步

- 用户首次进入无需登录即可使用。
- 任务先本地保存。
- 获取 openid 后同步到云端。
- 云同步失败不影响本地使用。
- 换设备后可通过 openid 恢复任务。
- 图片可通过 fileID 恢复展示。

### 19.7 周报

- 全部页右上角可进入周报。
- 周报展示本周完成数、未完成数、完成率。
- 周报按标签分组展示已完成任务。
- 周报展示待跟进任务。
- 用户可一键复制周报文本。
